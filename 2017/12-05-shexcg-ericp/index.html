Inheritance

Start with a general rule about a structure:

<ObservationShape> {
  …
  :component {
    :code . ;
    :value .
  } *
}

An observation contains N components, each with a code and a value.

Inheritance

Use that structure in a derivative shape:

<ObservationShape> {
  …
  :component {
    :code . ;
    :value .
  } *
}
<BP> -<ObservationShape> {
  …
  :component {
    :code [ "systolic" ] ;
    :value .
  };
  :component {
    :code [ "diastolic" ] ;
    :value .
  }
}

An BP observation contains 2 components with codes for systolic and diastolic.

Inheritance

Use that structure in turn in an extended shape:

<ObservationShape> {
  …
  :component {
    :code . ;
    :value .
  } *
}
<BP> -<ObservationShape> {
  …
  :component {
    :code [ "systolic" ] ;
    :value .
  };
  :component {
    :code [ "diastolic" ] ;
    :value .
  }
}
<PostureBP> &<BP> {
  …
  :component {
    :code [ "posture" ] ;
    :value .
  };
}

An PostureBP observation has an additional components with a code for posture.

Severity - single severity

<EventShape> {
  schema:startDate @<Date> ;
    // sx:severity sx:error
  schema:endDate @<Date> ;
    // sx:severity sx:warning
  schema:actor @<Person> ;
    // sx:severity sx:warning
}

<Person> {
  schema:name xsd:string
}
:hackathon1 {
  schema:startDate "2017-01-01"^^xsd:date ;
  schema:endDate "2017-01-01"^^xsd:date ;
  schema:actor :sue ;
}

:sue {
  schema:name "Sue"
}
:hackathon1 {
  schema:startDate "2017-01-01"^^xsd:date ;
  schema:endDate "2017-01-01"^^xsd:date ;
  schema:actor :sue ;
}

:sue {
  schema:name999 "Sue"
}
:hackathon1 {
  schema:startDate "2017-01-01"^^xsd:date ;
}

    // sx:max sx:warning

Severity - macros

HIGH { sx:value sx:error, sx:low sx:error, sx:high sx:error }
LOW { sx:value sx:error, sx:low sx:warning, sx:high sx:error }
